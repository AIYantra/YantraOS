# Security Policy

## Supported Versions

Currently, the following versions of YantraOS are supported with security updates.

| Version | Supported          |
| ------- | ------------------ |
| 1.0.x   | :white_check_mark: |
| 0.2.x   | :white_check_mark: |
| < 0.2.0 | :x:                |

## Reporting a Vulnerability

Security is a core pillar of YantraOS. Since YantraOS has the capability to execute root-level intents and operate autonomously, we take all vulnerabilities extremely seriously.

Please do **NOT** report security vulnerabilities via public GitHub issues.

Instead, report them privately to our security team via email:
**security@yantraos.com**

You should receive a response within 48 hours. If the issue is confirmed, we will release a patch as quickly as possible and credit you in our security advisories.

### What to include in your report:
- Type of issue (e.g., buffer overflow, prompt injection, IPC bypass, sandbox escape).
- Step-by-step instructions to reproduce the vulnerability.
- Proof-of-concept or exploit code (if available).
- The impact of the vulnerability.
